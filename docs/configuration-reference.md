# Configuration Reference

ClawSpec is configured via `clawspec.yaml`. The file is discovered by walking upward from the target directory until one is found. If no file is found, all settings revert to defaults.

## File Discovery

```
clawspec run skills/my-skill/tests/scenarios.yaml
```

ClawSpec searches for `clawspec.yaml` starting from `skills/my-skill/tests/` and walking upward through each parent directory until reaching the filesystem root. The first file found is used. This means you can have a root-level config that applies to all targets, overridden by a closer config for specific subtrees.

## All Fields

### Core fields

| Field | Type | Default | Description |
|---|---|---|---|
| `gateway_base_url` | string | `"http://127.0.0.1:18789"` | Base URL for the OpenClaw gateway. |
| `gateway_webhook_endpoint` | string | `"/webhook/mcp-skill-invoke"` | Endpoint path for skill invocations. |
| `gateway_auth_token` | string | `null` | Bearer token for gateway authentication. Load from env; do not hardcode. |
| `report_dir` | path | `"reports"` | Directory where test reports and baselines are written. Relative to `clawspec.yaml`. |
| `gateway_log_pattern` | string | `"/tmp/openclaw/openclaw-{date}.log"` | Pattern for gateway operational log files. `{date}` is replaced with `YYYY-MM-DD`. |
| `scenario_patterns` | list | `["skills/**/tests/scenarios.yaml", "agents/**/tests/scenarios.yaml"]` | Glob patterns used by `clawspec coverage` to discover scenario contracts. |
| `ledger_path` | path | `"coverage-ledger.yaml"` | Path to the coverage ledger file. Relative to `clawspec.yaml`. |

### Observability fields

These fields can be set directly in `clawspec.yaml` at the top level (for environment variable override compatibility) or under the nested `observability:` block (preferred for readability).

| Field | Type | Default | Description |
|---|---|---|---|
| `observability_backend` | string | `"none"` | Which backend to use. `"opik"` or `"none"`. |
| `trace_poll_delay_ms` | int | `3000` | Milliseconds to wait after a scenario run before querying for the trace. |
| `time_window_padding_ms` | int | `10000` | Extra milliseconds added to each end of the time window when searching for a trace. |
| `model_pricing` | dict | `{}` | Per-model cost table for client-side cost estimation. See example below. |
| `opik_project_name` | string | `""` | Opik project to scope trace searches. |
| `opik_workspace` | string | `""` | Opik workspace name. |

## Environment Variable Overrides

Every core field has a corresponding environment variable. Environment variables take precedence over `clawspec.yaml` values. Programmatic overrides (Python API `overrides=` argument) take precedence over environment variables.

| Environment variable | Field |
|---|---|
| `CLAWSPEC_GATEWAY_BASE_URL` | `gateway_base_url` |
| `CLAWSPEC_GATEWAY_WEBHOOK_ENDPOINT` | `gateway_webhook_endpoint` |
| `CLAWSPEC_GATEWAY_AUTH_TOKEN` | `gateway_auth_token` |
| `CLAWSPEC_REPORT_DIR` | `report_dir` |
| `CLAWSPEC_GATEWAY_LOG_PATTERN` | `gateway_log_pattern` |
| `CLAWSPEC_SCENARIO_PATTERNS` | `scenario_patterns` (comma-separated list) |
| `CLAWSPEC_LEDGER_PATH` | `ledger_path` |
| `CLAWSPEC_OBSERVABILITY_BACKEND` | `observability_backend` |

**Additional token fallbacks for `gateway_auth_token`:**

If `CLAWSPEC_GATEWAY_AUTH_TOKEN` is not set, ClawSpec also checks `HOOKS_TOKEN` and `OPENCLAW_HOOKS_TOKEN` in that order. This makes ClawSpec compatible with OpenClaw deployments that already have a hooks token configured.

**Observability secrets:**

`OPIK_API_KEY` is loaded by the Opik backend directly from the environment. It is never read from `clawspec.yaml`.

## Example: Complete Config File

```yaml
# clawspec.yaml

# Gateway
gateway_base_url: "http://127.0.0.1:18789"
gateway_webhook_endpoint: "/webhook/mcp-skill-invoke"
# gateway_auth_token: loaded from CLAWSPEC_GATEWAY_AUTH_TOKEN or HOOKS_TOKEN

# Output
report_dir: "reports"

# Logs
gateway_log_pattern: "/tmp/openclaw/openclaw-{date}.log"

# Coverage discovery
scenario_patterns:
  - "skills/**/tests/scenarios.yaml"
  - "agents/**/tests/scenarios.yaml"
ledger_path: "coverage-ledger.yaml"

# Observability (optional — remove block entirely to disable)
observability:
  backend: opik
  trace_poll_delay_ms: 3000
  time_window_padding_ms: 10000

  opik:
    project_name: "my-openclaw-project"
    workspace: "my-workspace"
    # api_key: NEVER here — use OPIK_API_KEY env var

  model_pricing:
    claude-sonnet-4-6:
      input_per_1k: 0.003
      output_per_1k: 0.015
    claude-haiku-4-5:
      input_per_1k: 0.00025
      output_per_1k: 0.00125
    claude-opus-4-6:
      input_per_1k: 0.015
      output_per_1k: 0.075
```

## Example: Compiler-Managed Repo

Repos where ClawSpec QA artifacts are generated by the compiler use overridden scenario patterns and ledger paths:

```yaml
# clawspec.yaml (compiler-managed repo)
scenario_patterns:
  - "compiler/generated/qa/**/scenarios.yaml"
ledger_path: "docs/testing/coverage-ledger.yaml"
```

## Python API Config

When using the Python API, you can load config from a specific file, pass a start directory for discovery, or inject programmatic overrides:

```python
from clawspec.config import ClawspecConfig

# Auto-discover from cwd
config = ClawspecConfig.load()

# Load from explicit path
config = ClawspecConfig.load(path="path/to/clawspec.yaml")

# Override fields programmatically (highest precedence)
config = ClawspecConfig.load(overrides={
    "observability_backend": "opik",
    "report_dir": "test-reports",
})

# Create a modified copy without mutating the original
dry_config = config.with_overrides(report_dir="dry-run-reports")
```

The `resolve_path()` method on `ClawspecConfig` resolves relative paths against `root_dir` (the directory containing `clawspec.yaml`):

```python
absolute_path = config.resolve_path("reports/baselines.yaml")
```

## Config Precedence (highest to lowest)

1. Programmatic `overrides=` argument
2. Environment variables
3. `clawspec.yaml` file values
4. Built-in defaults
